# 커밋 메시지 규칙 (저장소 표준)

## 어디가 기준인지

| 용도 | 위치 |
|------|------|
| **Cursor·Agent가 항상 볼 요약** | `.cursor/rules/commit-convention.mdc` (`alwaysApply: true`) |
| **이 문서** | 예시·톤·실무 안내를 길게 적을 때 |
| **진입점** | 루트 `.cursorrules` → 위 `.mdc` 목록 안내 |

`commit-convention.mdc`와 이 파일이 어긋나면 **먼저 `.mdc`를 고치고**, 여기서는 예시·설명만 맞춘다.

---

## 한 줄 요약

- 제목은 **보고체 말고** 짧고 직관적으로 (무엇이 어떻게 바뀌었는지 한눈에).
- 제목·본문 **한글**, 타입·scope·기술 고유명사만 영문.
- 형식: `<타입>(<범위>): <한글 제목 50자 이내 마침표 없음>`
- 여러 파일이면 본문에 bullet로 의도·변경 요약.

## User Rules (전역) 보조

다른 저장소에서도 비슷하게 맞추려면 Cursor **설정 → Rules for AI**에 예시:

```
[Git 커밋 메시지] 제목은 짧고 직관적으로(보고체 지양). 제목·본문 한글. 영문은 타입 태그·scope·고유명사만. 형식: <타입>(<범위>): <한글 50자 이내 마침표 없음>. 제목에 add/fix 같은 영문 동사 금지. 여러 파일이면 본문 bullet로 요약.
```

## Git 커밋 메시지 (상세)

**자동완성·생성·채팅 제안**할 때: 격식 있는 보고서 말고, **제목만 읽어도 “무슨 의미로 바뀌었는지”** 보이게 쓴다.

### 중점 (제목이 먼저)

- **사용자·운영 관점:** 동작이 어떻게 달라졌는지, 제약이 생겼는지.
- **리팩터/이름만 바뀐 커밋도** 제목에 **목적**을 넣는다. `파일 삭제 및 임포트 변경` 같은 **기계적 요약은 금지** — 그건 diff가 말해 주는 것.
- **본문 bullet:** 제목에 안 담은 **의도·주의사항·호환**을 먼저 쓰고, 그다음에만 모듈·파일 단위 구체를 붙인다. **삭제·임포트만 나열**하지 않기.

### 말투 (직관 우선)

- **이렇게:** 짧은 한 줄, 동료한테 말하듯. `~추가`, `~고침`, `~분리`, `~통일`, `이제 ~됨`, `~할 때 ~하도록`.
- **피하기:** `…에 대한 구현을 수행함`, `…을 명시적으로 처리함`, `…에 관한 개선을 반영함` 같이 딱딱한 문장.

### 꼭 지킬 것

- **제목·본문은 한글** (영문으로 통째로 쓰지 않음).
- **영문 OK:** 타입 `feat`/`fix`/… , scope(모듈명), 고유명사(Spring Boot, REST, JPA, React …).
- **제목에 영문 동사 금지:** `add`, `fix`, `update` 같은 건 제목에 넣지 않기.
- **형식:** `<타입>(<scope>): <한글 한 줄>` — 50자 안, 마침표 없음.

### 형식

```
<타입>(<범위>): <제목>

[필요할 때] 본문 bullet — 의도·주의 먼저, 파일 단위는 보조
```

### 제목 예시 (이 느낌)

- `feat(api): 게시글 목록에 페이지 크기 상한 걸어서 대량 조회 막음`
- `fix(auth): 만료된 리프레시 토큰이면 쿠키 지우고 로그인만 유도`
- `docs: README에 로컬 실행 순서랑 env 표 정리`
- `refactor(AiPlat): 공통 예외 응답은 한 핸들러만 타게 통일` ← 동작·목적이 보임
- `feat(ui): 결제 단계에서 뒤로 가도 입력값 유지되게 분리` ← 목적이 보임

### 본문 예시 (여러 파일)

```
feat(api): 역할별로 엔드포인트 접근 제한 넣고 관리자만 사용자 삭제 가능하게 함

- ADMIN만 DELETE /users/{id} 허용, 그 외는 403
- Security 설정·UserController·테스트에 동일 규칙 반영
```

### 피할 본문 (나쁜 예)

```
- UserService.java 삭제, AuthController.java에서 import 경로 변경
```

→ 제목에 **왜**를 넣거나, 본문 첫 줄에 **의도**(예: 인증 로직 단일 모듈로 모아 중복 제거)를 쓴다.
