## 2. 여러 부모를 가진 자식 노드 — UI **해결 불가 (의도적 보류)**

> **별도 코멘트:** README 8항목 전체를 만족시키려다 **전역 레이아웃 회귀**가 반복된 영역이다. **이번 마무리에서는 UI·레이아웃 솔루션을 제공하지 않는다.**

### 2.1 무엇이 문제인가

한 노드(또는 동일 논리 id)가 **부모를 여러 명** 가질 때, BDD에서 동시에 요구되는 것이 겹칩니다.

- **containment:** 부모 박스 **안**에 자식을 둠 (중첩)
- **specialization:** 부모 **위/아래** 타입 계층 + **빈 삼각형 엣지**
- **형제 배치:** 같은 컨테이너 안에서 **대칭·겹침 없음**

부모가 둘 이상이면 “한 자식을 어디에 그릴지”와 “상속선을 몇 개·어디에서 뽑을지”가 **한 좌표계**에서 충돌합니다.

### 2.2 케이스별 상태

| 케이스 | 예시 | 데이터 | UI·레이아웃 |
|--------|------|--------|-------------|
| **A. 다중 containment 부모** | 동일 `DataPort`가 `ECU`·`Sensor` 아래 | `BlockModelBuilder`: `id__in__parent` 복제 | ❌ 복제별 위치·spec 연결 **미구현** (롤백) |
| **B. 다중 spec 부모 (형제)** | test-8 `Gateway`→`Router`·`Firewall` | 원본 1노드 + spec 엣지 2개 | ❌ 부모마다 복제·아래 배치·짧은 선 **미구현** (롤백) |
| **C. 포함 부모 = spec 부모 (1명)** | test-9 `SCADA`⊂`GridController`, spec 동일 | `nestedSpecChrome` | ✅ 헤더 **🔼**, spec **선 숨김**, 선택 시 부모 HL |

**A·B는 “여러 부모 자식” UI 문제로 묶어 보류.** C만 **다른 렌더 모드**로 처리.

### 2.3 롤백·실패 이력 (다시 넣지 말 것)

아래를 넓게 넣으면 **test-1·test-4 전체 배치**가 깨진 사례가 있음 (`work-log` 참고).

- `packContainmentChildren` 가운데/왼쪽 정렬 변경
- `ensureContainmentParentsEncloseChildren` 전체 한 줄 재배치
- spec 자식을 가로 줄에서 전역 제외
- `expandMultiParentSpecClones` + 맨 마지막 `placeMultiParentSpecClones` (복제만 넣고 ELK/팩 순서 어긋남)
- `specEdgeRouter` 전역 최근접 면·형제 S 출구
- 헤더 🔼 + 접기 `precompute`/`spacingRight` 동시 확장 (겹침 이슈 잔존 시 롤백)

### 2.4 후속 시 권장 접근 (참고만)

1. **데이터:** 복제 노드·엣지 재배선만 PR (레이아웃 후처리 없음)  
2. **레이아웃:** `bddLayout` **맨 마지막** 단계에만 “복제 → 해당 spec 부모 아래” (pack/ensure 미변경)  
3. **렌더:** 복제 노드는 `MxCellFactory`에서 pack/겹침 대상 **제외**  
4. **엣지:** 복제→해당 부모만 spec 라우트; 원본 hidden spec은 제거

## 3. nested spec 크롬 (구현됨 — C케이스)

**조건:** `node.parent === specEdge.target` (직접 포함 부모가 곧 상속 부모)

| 항목 | 동작 |
|------|------|
| spec 엣지 | `MxEdgeBuilder` — `nestedSpecChrome` 이면 **셀 미생성** |
| 라벨 | `MxLabelUtils` — `🔼` + tooltip «포함 부모 타입 상속: …» |
| 선택 | `MxNeighborHighlight` — `nestedSpecParentIds` 부모 **연결 하이라이트** |
| 파이프라인 | `BlockModelBuilder` → `nodeTransformer` / `edgeTransformer` → `normalizer` + `nestedSpecChrome.js` |

**의도:** 박스 **안**은 containment, 타입 관계는 **헤더**로 읽게 함 (긴 U자 상속선 제거).
