## 3. 테스트별 “무엇을 개선해 달라”는 뜻

### test-1 (필수)

- Vehicle containment(`PowerTrain`→`Engine`…) **한눈에**
- `Car`/`Truck` **Vehicle 위·아래** specialization 계층
- `engine_p` 등 **푸터** 표시 (구현됨)
- README As-Is 5문제 전부

### test-2

- `Polygon`→`Shape`+`Drawable`+`Resizable` **다중 부모 spec**
- containment(`Canvas`→`Layer`→도형)와 spec 트리 **분리**

### test-8

- `Gateway`→`Router`·`Firewall` **다중 spec** — **UI 미해결(보류):** 부모마다 복제·각 부모 아래·짧은 선 (`07_마무리_및_알려진_한계.md` §2)

### test-3

- 조직 containment가 곧 구조 → **벤치마크** 후보
- association 15개는 **미표시 유지** (레이아웃 방해 방지)

### test-4

- `System→…→Primitive` 7단 **탑** + `SoftwareSystem`/`HardwareSystem` **별도 루트**
- `HWComponent`/`SWComponent`는 `Component` 자식인데 **탑 밖**
- 루트 겹침·Microcontroller 내부 겹침·긴 spec 선

### test-9

- `SCADA`⊂`GridController` + spec 동일 — **nested spec 크롬** 검증용 (🔼·선 숨김·선택 시 부모 HL)

### test-5 ~ test-10

- 공통: **루트·spec 많음** → 기본 밴드·겹침만 스모크
- test-6/7: **다중 상속** — test-8과 동일 **다중 부모 UI 보류**
- test-8~9: association 밀도 (화면 미표시, 하이라이트만)

## 5. 로드맵 (실행 순서)

> **2026-05-20 1차 마무리:** Phase 0~3·5는 **스모크·부분 반영**으로 종료. Phase 4(다중 부모 UI)는 **해결 불가 보류** (`07_마무리_및_알려진_한계.md` §2). 아래 체크는 **제출 전 자가 점검**용.

### Phase 0 — 기준선 (1~2일)

- [ ] README 8항목 체크리스트 + test-1 before/after
- [ ] F5 스모크: test-1 → test-4 → test-3
- [ ] test별 Pass / Partial / Fail 표 (`work-log` 또는 이 문서)

### Phase 1 — 골든 (test-1) 【필수】 — **1차 마무리 반영**

**1-A 노드 배치 (엣지 전제)** — 대부분 반영

- [x] containment ELK parent·padding 정합 (`layout.js` `_precomputedPaddingTop`)
- [x] specialization Y 밴드 (`applySpecializationVerticalLayout`)
- [x] Car / Truck → Vehicle **아래** 가로 정렬 (`applySpecializationAroundParentLayout`)
- [x] 캔버스 좌상단 이탈 보정 (`fitDiagramToMargins`)
- [x] 형제 겹침 (`resolveSiblingOverlaps`)
- [ ] F5 test-1: PowerTrain→Engine containment 한눈에·루트 분리 노드 감소

**1-B·1-C 엣지** — spec은 `specEdgeRouter` N→S, nested spec은 선 생략

- specialization 직교·앵커 (`specEdgeRouter` + `MxEdgeBuilder`)
