# 엣지 규칙
1. 동일한 출발지-목적지를 갖는 다중엣지는 선 중첩 및 단일 뼈대를 허용한다. ( 엣지 경로 단순성 / 선 교차 최소화 )
-> 출발지와 도착지가 명확하므로 겹쳐서 흐르는 걸 허용하되, 종단점에서 쪼개지게 한다.

2. 모든 엣지 경로는 모든 노드의 본체 영역 및 텍스트 영억을 물리적으로 밟고 지나가거나, 관통 할 수 없다.
-> 선이 이동하는 경로상에 노드가 있을 경우, 경계면 외곽에 지정된 최소 안전거리(Beffer Zone)을 두고, 바깥쪽 빈 공간 채널로 우회하여 흐르도록 강제한다.

3. 출발지나, 도착지가 다른 경우 이동하는 좌표가 같은 엣지들이 있을떄, 차선개념을 적용하여, 중첩되지않고 서로 개별적으로 움직일수있게해야함 
→ 구현: `MxEdgeBuilder` — 동일 (source,target) 다중엣지는 차선 분리 제외, 그 외 출발·도착 **중심점**이 `displaySettings.bdd.specLaneProximityPx` 이내면 `specLaneOffsetPx` 평행 분리. 같은 타깃(또는 같은 소스)으로 모이는데 소스(또는 타깃) 중심 가로 간격이 좁으면 N·S 앵커를 함께 분산(`specSiblingGap` 기반 임계).

4. 컨테이너 벽면 단일 포트 통과 정책 (Gateway Port)
내부에서 평행하게 정돈된 5 가닥의 선이 부모 박스의 경계선(테두리)을 뚫고 최종 탈출할 때는, 박스 테두리를 5군데나 난도질하며 뚫고 나가지 않는다.

부모 박스의 상단 테두리 중앙 또는 탈출 방향 면에 '가상의 통합 관문 포트(Gateway)'를 하나 지정하고, 5개의 선을 이 관문 직전에서 하나의 두꺼운 단일 버스 선으로 병합(Bundling)하여 부모 박스 외부로 통과시킨다.

부모 박스 외부로 빠져나간 이후에, 최종 목적지(최상위 부모 노드)에 도달하기 직전 단계에서 다시 차선이 쪼개지듯 분산(Distribution)시킨다.

5 외부 우회 전용 도로 확보 정책 (External Bypass Corridor)
부모 박스 내부에서 도저히 공간이 안 나와 박스 옆구리(동/서측)로 탈출해야 하는 선이 존재할 경우, 그 선은 부모 박스 외곽 경계선으로부터 일정 거리 이상 떨어진 외부 전용 우회 도로망으로만 다녀야 한다.

부모 박스 바로 옆에 바짝 붙어서 흐르다가 다른 영역에 있는 무관한 외딴섬 노드(예: Engine, Actuator 등)의 경계면을 침범하는 것을 금지하며, 무조건 다이어그램의 최외곽 마진 구역을 크게 돌아서 목적지로 향하도록 직각 경로를 강제한다.


# 상속 관계 (specialization) 
1. 계층 배치: 부모(위) - 자식(아래) (계층 적 배치)
2. 앵커 배치: 부모(하단) - 자식(상단) (엣지 경로 단순성)
-> 앵커를 옆,위아래 무분별하게 할경우 계층구조 가독성을 잃을수있음. 확실하게 부모-자식관계라는것을 위해 강제.


# 포함관계
1. 포함 관계(containment)는 2개 이상일 때 선을 그리지 않고 '박스 중첩(Nesting)'으로만 표현한다. (선 교차 최소화)
-> 포함관계일 경우 자식을 부모 박스 내부 컨테이너로 내장하는 기법(Nesting)으로 배치한다. (계층적 배치)
-> 포함관계 자식노드가 1개뿐일때는 단순 아래방향 화살표로 단일노드 끼리 잇는 방식으로 표현
2. 형제노드들을 바둑판식 대칭 배치 적용 (대칭성 / 균일 간격 / 노드겹침 없음)
-> 좌우 대칭 균등하게 유지
3. 서로 다른 부모를 가진 노드가 있다면 각 부모의 포함관계로 표시. 

## 직각 강제 규칙
1. 엣지 경로가 직선(2점)일때도 최소 1개의 강제 경유점(WayPoint)를 주입하여 직각 강제 (엣지 경로 단순성)
-> mxGraph는 경로 점이 시작-끝 2개뿐이라면, 옵션이 있어도 대각선 지름길을 그리는 속성이 있음.
이를 방지하기위해 두 노드가 완벽한 일직선상에 있지않다면 강제로 직각으로 꺾이는 중간 좌표를 1개 연산하여 주입.

2. 모든 선은 수평선과 수직선의 조합으로만 구성되어야한다. (대각선,사선을 예외없이 금지한다.)



## 노드 배치 규칙

1. 노드별 충분한 거리를 주어서 엣지가 다른 노드 뚫는것 방지 (선 교차 최소화 / 노드 관통 방지)
-> 각 노드별 크게 여분 공간을 주어 위아래로 화살표가 나와도 각자의 노드를 뚫지않고, 여유있게 탈출할수있게하기위함.
