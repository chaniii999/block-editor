# SELab Block Editor — 간단 지도

VS Code 확장이 JSON을 읽어 WebView에서 **ELK(배치·직교 라우팅) + mxGraph(그리기)**로 블록 다이어그램을 보여 주는 구조입니다.

---

## 디렉터리 한눈에

| 경로 | 역할 |
| --- | --- |
| `src/extension.js` | 커맨드 등록, 확장 활성화 |
| `src/BlockDiagramPanel.js` | WebView 패널, JSON 기준 모델 로드·`postMessage` |
| `src/HtmlGenerator.js` | WebView HTML 조립 (`media`·번들 URI) |
| `src/panel/index.js` | `fetchBlockModel` = 로드 + `buildBlockModel` |
| `src/panel/LanguageServerBridge.js` | 다이어그램 JSON 로드 |
| `src/panel/BlockModelBuilder.js` | 블록 뷰용 노드/엣지 필터·`nestedSpecChrome`·`__in__` 복제 |
| `src/panel/PanelMessageHandler.js` | WebView ↔ 확장 메시지 처리 |
| `media/editor/model/nestedSpecChrome.js` | 웹뷰 normalize 시 포함=상속 부모 크롬 플래그 |
| `media/editor/layout.js` | ELK 전 `precomputeNodeSizes`(헤더·🔗·접기 폭) |
| `media/editor/layout/bddLayout.js` | ELK 후 spec 밴드·형제·겹침 |
| `media/editor/mxgraph/specEdgeRouter.js` | specialization 선 라우팅 (N→S) |
| `media/editor/mxgraph/MxNeighborHighlight.js` | 선택·연관·연결·nested spec 부모 하이라이트 |
| `dist/extension.js` | esbuild 출력 — 실제 로드되는 확장 진입점 |
| `media/editor/boot.js` | WebView 부트, ELK 적용, mxGraph 렌더 연동 |
| `media/editor/layout/elkLayout.js` | ELK 그래프·옵션·레이아웃 |
| `media/editor/config/displaySettings.js` | ELK·간격·노드 크기 등 설정 집약 |
| `media/editor/mxgraph/` | mxGraph 래퍼·셀 팩토리 등 |
| `packages/selab-ui/` | 툴바·공통 UI (로컬 패키지) |
| `tests/*.json` | 렌더링용 픽스처 데이터 |
| `scripts/build.mjs` | `src/extension.js` → `dist` 번들 |

## 데이터 흐름도

### 상세 (컴포넌트 단위)

```mermaid
flowchart LR
  subgraph host["확장 호스트 Node"]
    EXT[extension.js]
    PNL[BlockDiagramPanel.js]
    FETCH[panel/index.js]
    LSB[LanguageServerBridge.js]
    BMB[BlockModelBuilder.js]
    HTML[HtmlGenerator.js]
  end

  subgraph webview["WebView 브라우저"]
    BOOT[boot.js]
    ELK[elkLayout.js]
    MX[mxgraph / render]
    DS[displaySettings.js]
  end

  JSON[(tests/*.json)]
  USER[사용자 F5 / 우클릭 열기]

  USER --> EXT
  EXT --> PNL
  JSON --> LSB
  PNL --> FETCH
  FETCH --> LSB
  FETCH --> BMB
  PNL --> HTML
  HTML --> BOOT
  PNL -->|postMessage model| BOOT
  BOOT --> ELK
  ELK -.->|옵션 참조| DS
  BOOT --> MX
  ELK -->|좌표·엣지 경로 반영| MX
```

### 단계 요약 (순서만)

```mermaid
flowchart TD
  A[JSON 파일 on disk] --> B[LanguageServerBridge: 읽기]
  B --> C[BlockModelBuilder: 블록 뷰 모델]
  C --> D[BlockDiagramPanel: postMessage]
  D --> E[WebView boot: normalizeModel]
  E --> F[precomputeNodeSizes + applyElkLayout]
  F --> G[bdd.applyPostLayout]
  G --> H[mxGraph renderModel + reroute edges]
```

## 빌드·실행 한 줄

1. `npm install` → `npm run build` (`dist/extension.js` 생성)  
2. VS Code에서 **F5** → Extension Development Host  
3. `tests/test-1.json` 열기 → **블록 다이어그램 옆에 열기**
